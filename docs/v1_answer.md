18. 낙찰 후 **배송**은 어떻게 처리하나요? (실물 배송 / 디지털 파일 / 둘 다)
=> 작가가 실물 배송
18-1. 배송 분쟁 (분실/파손) 책임 소재
=> 배송 프로세스는 플랫폼 시스템에 포함되지 않음
=> 분쟁은 작가-구매자 간 직접 처리 (플랫폼 미개입)
=> 단, 신고 접수 시 제재 시스템에 반영 가능
18-2. 낙찰 후 미결제 처리 (일반 경매 시스템 프로세스 참고)
=> 1) 낙찰 후 결제 기한 설정 (기본 3일, 시스템 설정 가능)
=> 2) 자동 알림 발송 (낙찰 즉시, 1일 전, 마감일)
=> 3) 기한 초과 시 낙찰 자동 취소
=> 4) 차순위 입찰자에게 낙찰 기회 이전 (Second Chance Offer)
=> 5) 차순위 없거나 거절 시 재경매 등록
=> 6) 미결제 낙찰자 제재: 경고 누적 → 경매 참여 제한 (I-41 경고 시스템 연계)
19. **디지털 아트(NFT, AI 생성 등)** 는 다루지 않는다고 하셨는데 픽스인가요?
=> 디지털 아트는 취급하지 않음
=> 작품 업로드 시 시스템 관리자가 수동 판독 (AI 자동 판독 미사용)
=> 의심 작품은 관리자 검토 큐에 자동 등록

45. 미성년 작가/유저 정책
=> 국제 법규 준수 (각 국가별 디지털 서비스 최소 연령 기준 적용)
=> 주요 기준:
   - EU(GDPR-K): 16세 미만 보호자 동의 필요 (국가별 13~16세 차등)
   - 미국(COPPA): 13세 미만 보호자 동의 필수
   - 한국: 14세 미만 보호자 동의 필수
=> 미성년 작가: 보호자 동의 + 보호자 명의 정산 계좌 필수
=> 미성년자 결제: 보호자 연동 계정 또는 선불 한도 제한

46. 국가별 세금 처리 가이드라인
=> 기본 원칙: Stripe Tax, Avalara 등 **세금 자동화 서비스 연동** 권장
=> 작가 정산 시 **국가별 원천징수/세무 폼** 자동 발행
=> 국가별 주요 기준:

| 국가 | 작가 소득 처리 | 플랫폼 거래 과세 |
|------|----------------|-------------------|
| 한국 | 3.3% 원천징수 (사업소득) | 부가세 10% (간이/일반 구분) |
| 미국 | 1099-K 발행 (연 $600 이상) | 주별 Sales Tax 상이 |
| EU | 작가 개별 신고 | VAT MOSS (구매자 국가 VAT) |
| 영국 | Self Assessment | VAT 20% (연 £85K 초과 시) |
| 일본 | 원천징수 불필요 | 소비세 10% (간이과세 1천만엔 기준) |
| 캐나다 | T4A 발행 | GST/HST 5~15% |
| 호주 | ABN 기반 | GST 10% |

=> 작가에게 **인보이스 자동 생성** 기능 제공
=> 연 1회 세무 리포트 다운로드 제공 (1099-K, T4A 등)
=> 정책 공지: "세금 신고는 작가 본인 책임" 명시

47. 디지털 아트 판독 방식
=> 시스템 관리자가 수동 판독 (D-19 참조)

# 고객 요구사항 vs 현재 구현 Gap 분석

**Date**: 2026-04-13
**Source**: 질의서 1차 고객 답변
**Scope**: 전체 플랫폼 기능 A~J 영역

---

## 핵심 변경 사항 요약

> **플랫폼 명칭**: Domo Lounge (확정)
> **UI 방향**: 넷플릭스형 큐레이션 + 순위 피드 혼합 (현재 X 스타일에서 전환 필요)
> **블루버드 단가**: ₩1,000 → **$1 USD** 변경 필요
> **DM**: 불가 (게시판/게시글로 대체)
> **디지털 아트**: 완전 제외 확정

## A. 유저 경험 & 온보딩

### A1. 작가 인증 방식 강화

| 항목 | 현재 구현 | 고객 요구 | Gap | 우선순위 |
|---|---|---|---|---|
| 본인인증 | ❌ 없음 (Google OAuth만) | ✅ 필수 (작가/컬렉터 모두) | 🔴 **신규** | P1 |
| 학교 이메일 인증 | ❌ 없음 | ✅ `.edu` 이메일 인증 | 🔴 **신규** | P1 |
| 재학/졸업 증빙 | ❌ 없음 | ✅ 증빙 서류 업로드 | 🔴 **신규** | P1 |
| 졸업 후 자동 전환 | ❌ 없음 | ✅ 학생 → 신진작가 자동 전환 | 🔴 **신규** | P2 |
| 포트폴리오 검토 | ✅ `portfolio_urls`, `sample_images` 존재 | 일치 | ✅ | - |

**수정 필요**: 
- `ArtistApplication` 모델에 `edu_email`, `enrollment_proof_url`, `graduation_year`, `is_graduated` 필드 추가
- 이메일 인증 플로우 (`.edu` 도메인 검증 + 인증 코드)
- 본인인증 연동 (PASS/Toss 등 외부 서비스) — 별도 PDCA 사이클

### A2. 작가 프로필 필드 확장

| 항목 | 현재 구현 | 고객 요구 | Gap |
|---|---|---|---|
| 이름/활동명 | ✅ `display_name` | 일치 | ✅ |
| 소속 학교 | ✅ `school` | 일치 | ✅ |
| 학과 | ❌ 없음 | ✅ 필수 | 🟡 추가 |
| 졸업연도/재학 여부 | ❌ 없음 | ✅ 필수 | 🟡 추가 |
| 대표 작품 3~6개 | ⚠️ `sample_images` (URL 배열) | 작품명+설명+사이즈+재질 필요 | 🟡 구조 변경 |
| 자기소개 200자 | ✅ `statement` | 제한 없음 → 200자 제한 추가 | 🟢 수정 |
| 장르/스타일 태그 | ⚠️ Post 레벨에만 | 프로필 레벨에도 필요 | 🟡 추가 |
| 전시/수상 이력 | ❌ 없음 | ✅ 선택 항목 | 🟡 추가 |

**수정 필요**:
- `ArtistProfile`에 `department`, `graduation_year`, `is_enrolled`, `genre_tags`, `exhibitions`, `awards` 필드 추가
- `sample_images` → 구조화된 `representative_works` (JSON array of objects)

### A3. 사용자 유형 분리

| 항목 | 현재 구현 | 고객 요구 | Gap |
|---|---|---|---|
| 유저 role | ✅ `user / artist / admin` | 일치 | ✅ |
| 컬렉터 역할 구분 | ❌ `user`로 통합 | `collector` 역할 분리 희망 | 🟡 검토 |
| 컬렉터 본인인증 | ❌ 없음 | ✅ 금융거래 발생으로 필수 | 🔴 **신규** |

**의견**: `collector`를 별도 role로 분리하기보다 `user.is_verified` 플래그로 본인인증 여부를 관리하는 것이 간결. 금융거래(구매/후원) 시 `is_verified=true` 요구.

| 카테고리 큐레이션 | ❌ 장르 칩 필터만 | ✅ 장르별 가로 카드 행 | 🔴 **신규** |

**View 모드 전환 제안** (현재 방식 유지 + 넷플릭스 모드 추가):

```
┌─ View 모드 토글 ──────────────────┐
│  [리스트 뷰]  [갤러리 뷰]         │
│   현재 X 스타일   넷플릭스 스타일  │
└───────────────────────────────────┘
```

- **리스트 뷰 (현재)**: 피드 리스트 + 우측 레일. 팔로잉 기반, SNS 느낌

비로그인 기본값 = 갤러리 뷰, 로그인 기본값 = 리스트 뷰 (설정 가능).

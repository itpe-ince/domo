## 1. 현재 구현된 관리자 기능

### 백엔드 API (4파일, 14개 엔드포인트)

| 카테고리 | 엔드포인트 | 기능 |
|---|---|---|
| 작가 심사 | `GET /admin/artists/applications` | 심사 신청 목록 |
| | `POST .../approve` | 승인 (role 변경 + 프로필 생성 + 알림) |
| | `POST .../reject` | 거부 + 알림 |
| 콘텐츠 판독 | `GET /admin/posts/digital-art-queue` | 판독 대기 포스트 |
| | `POST /admin/posts/{id}/digital-art-verdict` | 판독 승인/거부 |
| 경매 | `POST /admin/auctions/process-expired` | 만료 주문 처리 |
| 모더레이션 | `GET /admin/reports` | 신고 목록 |
| | `POST /admin/reports/{id}/resolve` | 신고 처리 (경고/기각) |
| | `GET /admin/appeals` | 이의 제기 목록 |
| | `POST /admin/warnings/{id}/cancel` | 경고 취소 |
| | `POST /admin/warnings/{id}/reject-appeal` | 이의 기각 |
| 대시보드 | `GET /admin/dashboard/stats` | 유저/콘텐츠/매출 통계 |
| | `GET /admin/dashboard/revenue` | 매출 상세 (소스별) |
| 설정 | `GET/PATCH /admin/settings/{key}` | 시스템 설정 CRUD |

### 프론트엔드 페이지 (4개)

| 페이지 | 경로 | 현재 상태 |

| 모더레이션 | `/admin/moderation` | ✅ 신고 + 이의 제기 |
| 시스템 설정 | `/admin/settings` | ✅ key-value 에디터 |

## 2. 고객 요구사항에서 도출된 관리자 기능

### A. 유저 & 작가 관리

| # | 기능 | 현재 | 필요 | 우선순위 |
|---|---|---|---|---|
| A1 | **유저 목록/검색** | ❌ | 이름/이메일/역할/상태 필터 + 검색 | P1 |

## 3. 관리자 메뉴 구조 (최종)

```
📊 관리자
├── 대시보드 ────────── 통계 + 매출 + 차트 (✅ 구현됨)
│
├── 유저 관리
│   ├── 유저 목록 ──── 검색/필터 + 상세 + 정지/복구      (🔴 P1)
│   ├── 작가 심사 ──── 신청 목록 + 확장 필드 + 승인/거부   (✅+확장)
│   └── 등급 관리 ──── 배지 수동 변경 + 등급 분포          (🟡 P2)
│
├── 학교 관리 ──────── 학교 CRUD + 도메인 인증 + 국가별 필터 (🔴 P1)
│
├── 콘텐츠 관리
│   ├── 포스트 목록 ── 검색/필터 + 숨김/삭제               (🔴 P1)
│   ├── 판독 큐 ────── 디지털 아트 판독 승인/거부           (✅ 구현됨)
│   └── 컬렉션 ────── 시리즈 목록 + 검수                   (🟢 P3)
│
├── 거래 관리
│   ├── 경매 현황 ──── 진행/종료/정산 대기                  (🔴 P1)
│   ├── 주문 목록 ──── 전체 주문 + 상태 필터               (🔴 P1)
│   ├── 정산 관리 ──── 정산 대기/실행/환불                  (🔴 P1)
│   └── 후원 내역 ──── 후원/구독 목록                      (🟡 P2)
│
├── 모더레이션
│   ├── 신고 처리 ──── 신고 목록 + 처리                    (✅ 구현됨)
│   ├── 이의 제기 ──── 경고 취소/기각                      (✅ 구현됨)
│   └── 제재 이력 ──── 유저별 전체 이력                    (🟡 P2)
│
├── 분석
│   ├── 검색 트렌드 ── 인기 검색어 + 추이                  (🟡 P2)
│   ├── 등급/국가 분포 차트                                (🟡 P2)
│   └── 전환율 퍼널 ── 가입→신청→승인→판매                (🟢 P3)
│
└── 시스템 설정 ────── 수수료/단가/제한값 편집              (✅+확장)
```

## 6. Next Step

관리자 패널은 범위가 크므로 2가지 진행 방식 제안:

**A. artist-onboarding과 합치기**: Phase 1에 학교 관리 + 작가 심사 확장만 포함

**B. 별도 PDCA 사이클**: `/pdca plan domo-admin-panel`로 분리하여 유저/콘텐츠/거래 관리를 체계적으로 진행

추천: **A 방식** — artist-onboarding Phase 1에 학교 관리(B1~B3) + 작가 심사 확장(A7)을 포함하고, 나머지 P1 항목(유저/콘텐츠/거래 관리)은 별도 사이클로 분리.

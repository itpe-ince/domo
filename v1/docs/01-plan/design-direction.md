## 1. 디자인 컨셉

### 1.1 비주얼 모티브: "두쫀쿠" (두바이 쫀득 쿠키)

두바이 쫀득 쿠키의 비주얼에서 영감을 받은 컬러 테마입니다.

- **겉(외피)**: 다크 초콜릿 — 깊고 진한 다크 브라운 톤
- **속(필링)**: 피스타치오 그린 — 밝고 선명한 생동감 있는 녹색

이 대비가 Domo의 시각적 정체성을 만듭니다. 다크한 배경 위에서 **피스타치오 그린 액센트**가 작품과 주요 인터랙션을 돋보이게 합니다.

### 1.2 톤앤매너

| 키워드 | 설명 |
|--------|------|
| Dark & Premium | 다크 테마 기반, 프리미엄 갤러리 느낌 |
| Immersive | 작품이 주인공, UI는 배경에 녹아듦 |
| Bold Accent | 피스타치오 그린의 과감한 강조 |
| Minimal & Clean | 장식 요소 최소화, 여백 활용 |
| Warm | 완전한 블랙/화이트가 아닌 웜 계열 |

### 1.3 참고 레퍼런스

| 요소 | 레퍼런스 | 활용 포인트 |
|------|----------|-------------|
| 전체 다크 테마 | X (Twitter) | 네비게이션, 피드 레이아웃, 사이드바 |
| 몰입형 콘텐츠 | TikTok | 작품 상세/전체 화면 뷰, 세로 스와이프 |
| 작품 그리드 | Instagram Explore | 장르별 탐색 화면 |
| 경매 UI | (자체 커스텀) | 입찰 실시간성, 카운트다운 강조 |

## 2. 컬러 팔레트

### 2.1 핵심 컬러

| 역할 | Hex | 용도 |
|------|-----|------|
| **Background** | `#1A1410` | 메인 배경 (다크 초콜릿) |
| **Surface** | `#2A2018` | 카드, 모달, 네비게이션 바 |
| **Surface Hover** | `#352821` | 인터랙션 시 밝아짐 |
| **Border** | `#3D2F24` | 구분선, 카드 테두리 |
| **Primary** | `#A8D76E` | 버튼, 뱃지, CTA, 블루버드 후원 (피스타치오 그린) |
| **Primary Hover** | `#BDE284` | 버튼 호버 상태 |
| **Primary Muted** | `#5E7A3E` | 비활성 상태, 보조 사용 |
| **Text Primary** | `#F5EFE4` | 본문 텍스트 (크림 오프화이트) |
| **Text Secondary** | `#B5A99A` | 보조 텍스트 |
| **Text Muted** | `#7A6F60` | placeholder, 약한 텍스트 |
| **Danger** | `#E85D5D` | 경고, 제재, 에러 |
| **Warning** | `#F0B14A` | 주의 알림 |
| **Success** | `#A8D76E` | 성공 (Primary와 통일) |

> 위 Hex는 **초안**이며, Design 단계에서 접근성 검증(WCAG AA 대비비 4.5:1) 후 최종 확정합니다.

### 2.2 컬러 사용 원칙

- **Primary 사용은 절제**: 한 화면에 1~2개의 Primary 요소만. 과하면 강조 효과 상실.
- **작품 컨텐츠 우선**: UI 색상은 배경 역할, 작품 이미지가 가장 생동감 있어야 함.
- **뱃지와 알림**: 블루버드 후원, 작가 인증, 낙찰 성공 등 주요 상태는 Primary 사용.
- **다크 전용**: 프로토타입은 라이트 모드 미지원. 필요시 2차 출시에 추가.

## 4. 레이아웃 원칙

### 4.1 반응형 브레이크포인트

| 디바이스 | 브레이크포인트 |
|----------|----------------|
| Mobile | < 640px |
| Tablet | 640px ~ 1024px |
| Desktop | 1024px ~ 1440px |
| Wide | ≥ 1440px |

### 4.2 주요 화면 구조

- **데스크톱 피드**: X 스타일 3컬럼 (네비 | 피드 | 추천/트렌드)
- **모바일 피드**: 상단 탭 네비 + 세로 피드 + 하단 바텀 탭
- **작품 상세**: TikTok 스타일 전체 화면 몰입 뷰 (이미지/영상 + 오버레이 UI)
- **경매 상세**: 작품 이미지 중심 + 실시간 입찰 패널 (카운트다운 강조)

### 4.3 여백 시스템

```
spacing: 4 / 8 / 12 / 16 / 24 / 32 / 48 / 64 px
```

## 5. 컴포넌트 스타일 가이드

### 5.1 버튼

| 타입 | 스타일 |
|------|--------|
| **Primary** | Primary 배경 + 다크 텍스트, 강한 대비 |
| **Secondary** | Surface 배경 + 크림 텍스트 + Primary 테두리 |
| **Ghost** | 배경 없음 + 크림 텍스트, 호버 시 Surface 배경 |
| **Danger** | Danger 배경 + 화이트 텍스트 |

- 둥근 모서리: `rounded-full` (알약형) 또는 `rounded-lg` (8px)
- 기본 패딩: `px-5 py-2.5`

### 5.2 뱃지 (Badge)

- 피스타치오 Primary 배경 + 다크 텍스트
- 용도: 작가 인증, 블루버드 후원자, 경매 진행 중, 낙찰 완료
- 작고 둥근 형태, 아이콘 + 짧은 텍스트

### 5.3 카드 (Post/Product Card)

- 배경: Surface
- 테두리: Border 색상 (얇게)
- 호버: Surface Hover로 전환 + 부드러운 그림자
- 둥근 모서리: `rounded-xl` (12px)
- 작품 이미지: 카드 전체 너비, 비율 유지

### 5.4 입력 필드

- 배경: Surface
- 테두리: Border, 포커스 시 Primary
- 텍스트: Text Primary
- Placeholder: Text Muted

### 5.5 네비게이션

- **데스크톱 사이드바**: X 스타일, 아이콘 + 라벨
- **모바일 바텀탭**: TikTok 스타일, 5개 탭 (피드 / 탐색 / 업로드 / 알림 / 프로필)

## 6. 인터랙션 & 모션

- **전환**: 200~300ms, `ease-out`
- **호버**: 색상 전환 + 미세한 스케일(1.02)
- **경매 카운트다운**: 마지막 10초 펄스 애니메이션
- **블루버드 후원**: 성공 시 피스타치오 파티클 이펙트
- **로딩**: 스켈레톤 UI (다크 톤에 맞춘 Surface 색)

## 7. 구현 스택

| 항목 | 기술 |
|------|------|
| CSS Framework | Tailwind CSS v4 |
| 컴포넌트 라이브러리 | shadcn/ui (다크 테마 커스터마이징) |
| 아이콘 | Lucide React |
| 애니메이션 | Framer Motion (선택적) |
| 다크 모드 | 기본값 (라이트 모드 미지원) |

### 7.1 Tailwind 커스텀 설정 (초안)

```js
// tailwind.config.ts
theme: {
  extend: {
    colors: {
      background: '#1A1410',
      surface: {
        DEFAULT: '#2A2018',
        hover: '#352821',
      },
      border: '#3D2F24',
      primary: {
        DEFAULT: '#A8D76E',
        hover: '#BDE284',
        muted: '#5E7A3E',
      },
      text: {
        primary: '#F5EFE4',
        secondary: '#B5A99A',
        muted: '#7A6F60',
      },
      danger: '#E85D5D',
      warning: '#F0B14A',
    },
    fontFamily: {
      sans: ['Pretendard', 'Inter', 'sans-serif'],
    },
  },
}
```
